## Intent
The visible event in a channel: creating a planning event with `/plan`, the live-updating event message (embed + action buttons), and participation (Accept / Leave). After this epic a group can create an event and opt in; the message reflects the current participant list. Availability input and results come in later epics.

## Features
- `FEAT-event-lifecycle-1` — `/plan` create-event command & validation
- `FEAT-event-lifecycle-2` — Event message (embed + action buttons) with live update
- `FEAT-event-lifecycle-3` — Participation (Accept / Leave)
